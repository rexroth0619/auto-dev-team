# Verification Checklist

> Optional. Use when the user asks to validate the result explicitly.

## PM Validation

- [ ] Core behavior works
- [ ] Boundary cases behave correctly
- [ ] Key user flows work
- [ ] UX is acceptable

## Agent Validation

- [ ] No lint errors
- [ ] Backend tests actually ran
- [ ] GUI validation state is explicit
- [ ] GUI-capable tasks recorded executor and evidence
- [ ] Business questions are tracked
- [ ] Residual risk is documented
- [ ] Project docs were updated when needed
- [ ] Archive exists

## Regression Validation

- [ ] Existing behavior still works
- [ ] Related pages and features still work
