# 标准比喻模板：物流 / 仓储配送

## 适用场景

- 解释数据流
- 解释异步任务、队列、通知
- 解释状态同步失败
- 解释链路中间哪一段断了

## 不适用场景

- 纯 UI 细节和体验表述
- 极度强调产品语气的轻量解释

## 核心映射

| 技术对象 | 物流比喻 |
|----------|----------|
| 用户请求 | 发货单 / 配送请求 |
| Brainstorm | 确认寄什么、寄到哪、时效要求 |
| Architect | 设计运输路线和分拣流程 |
| Step | 分拣、装车、配送的分段执行 |
| Backend | 分拣中心 / 配送系统 |
| Frontend | 收件查询页 / 通知页 |
| Database | 仓储账本 |
| Queue / Async | 待派送队列 |
| Auth / Permission | 收件身份核验 |
| Cache | 中转暂存站 |
| Test / Validation | 抽检 / 模拟配送 |
| Release / 验收 | 试跑配送链路 |

## 常见故障映射

- 包裹生成了但没派送
  - 技术含义：任务创建了，但后续执行链没触发
- 到站了但没通知
  - 技术含义：后端状态更新了，但通知/前端没同步
- 签收了但状态没回传
  - 技术含义：结果落地了，但反向同步失败

## 用户常见问法

- 包裹卡在哪一段了？
- 为什么用户还没收到？
- 能不能给这一步加一个专门配送节点？

## 推荐输出风格

- 先说“现在在配送链路的哪一段”
- 再补技术链路和风险
