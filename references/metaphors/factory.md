# 标准比喻模板：工厂 / 流水线

## 适用场景

- 解释计划拆步
- 解释工程执行顺序
- 解释测试、质检、返工、发布验收
- 解释“为什么不能一口气全改”

## 不适用场景

- 纯用户体验类细节
- 强调轻松口语化沟通的场景

## 核心映射

| 技术对象 | 工厂比喻 |
|----------|----------|
| 用户请求 | 产品需求单 |
| Brainstorm | 确认规格、边界、验收标准 |
| Architect | 设计生产线和工序 |
| Step | 工位逐步加工 |
| Backend | 生产工段 |
| Frontend | 成品展示与交付面 |
| Database | 生产账本 / 物料系统 |
| Queue / Async | 待加工批次 |
| Auth / Permission | 工位权限 / 开工许可 |
| Cache | 半成品暂存区 |
| Test / Validation | 质检 |
| Release / 验收 | 出厂检验 / 试生产 |

## 常见故障映射

- 半成品做好了但没流到下一工位
  - 技术含义：中间层产物已生成，但流程没有继续推进
- 成品质检通过了但包装展示没更新
  - 技术含义：后端处理和验证都通过了，但展示层没同步
- 某个工位总返工
  - 技术含义：当前步骤设计或实现不稳定，需要重构或补测试

## 用户常见问法

- 现在卡在哪个工位？
- 能不能先只完成前两个工位？
- 为什么质检过了还不能出厂？

## 推荐输出风格

- 先说当前流转到哪一道工序
- 再补技术步骤、验证结论与剩余风险
