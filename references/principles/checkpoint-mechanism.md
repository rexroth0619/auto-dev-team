# 版本保护机制

> 三层保护体系：里程碑 🎯 / 保护快照 💿 / 存档 💾，确保可回退、可追溯

## 核心概念

| 术语 | 图标 | 定义 | 存储 |
|------|------|------|------|
| **里程碑** | 🎯 | 核心版本锚点，标记任务边界 | commit + git tag |
| **保护快照** | 💿 | 写代码前的自动现场保护 | commit |
| **存档** | 💾 | 每步改动完成后的记录 | commit |
| **指纹** | - | ≤10 字的业务功能摘要 | commit message |
| **工作分支** | - | AI 干活用的本地分支，与主分支隔离 | - |
| **受保护分支** | - | 不允许直接操作的分支（如 main/master/production） | - |
| **集成分支** | - | 功能完成后要合并到的目标分支 | - |

## 分支安全规则

### 1. 受保护分支

从 `.autodev/path.md` 读取 `protected_branches`，默认：
- `main`
- `master`
- `production`
- `release/*`

### 2. 集成分支（合并目标）

优先级：
1. `.autodev/path.md` 中的 `integration_branch`
2. `origin/HEAD` 指向的分支
3. 任务起始时所在分支

### 3. 工作分支命名

```
autodev/<任务简述>-<MMDD>
```

示例：`autodev/会员登录-0307`

### 4. 任务启动时的分支守卫

AI 开始任何代码改动前，**必须执行**：

```
检查当前分支
├─ 在受保护分支 → 自动创建工作分支，输出提示
└─ 在工作分支 → 继续
```

**创建工作分支后输出**：
```
━━━━━━━━━━━━━━━━━━━━
🌿 已创建工作分支
分支: autodev/会员登录-0307
基于: main (a1b2c3d)
━━━━━━━━━━━━━━━━━━━━
```

## 🎯 里程碑

里程碑是最重要的版本锚点，代表"可信赖的安全回退点"。保持稀缺，一个任务通常只有 1-2 个。

### 触发条件（只有 4 种）

| # | 触发点 | 条件 |
|---|--------|------|
| 1 | **任务开始** | 进入任何写模式时，在 write-preflight 中建立 |
| 2 | **信任模式开始** | 用户选择信任模式执行前 |
| 3 | **任务完成** | 用户确认任务完成时（可选，询问用户） |
| 4 | **用户指定** | 用户说"建立里程碑" |

### 指纹规范

- `{业务摘要}#起点` — 任务开始
- `{业务摘要}#信任起点` — 信任模式开始
- `{业务摘要}#完成` — 任务完成
- `{用户指定名}` — 用户自定义

### Commit + Tag 格式

```bash
git add -A
git commit -m "「{指纹}」chore: {描述}"
git tag milestone/{任务简述}-{MMDD}
```

示例：
```bash
git commit -m "「会员登录#起点」chore: 任务开始前基线"
git tag milestone/会员登录-0312
```

### 输出格式

```
━━━━━━━━━━━━━━━━━━━━
🎯 里程碑
━━━━━━━━━━━━━━━━━━━━
指纹: {任务}#起点
哈希: c3d4e5f
标签: milestone/{任务}-0312
━━━━━━━━━━━━━━━━━━━━
💡 随时可说「回退到 {任务}#起点」
━━━━━━━━━━━━━━━━━━━━
```

## 💿 保护快照

保护快照是写代码前的自动安全网，保留"改动前"状态，防止意外代码丢失。

### 智能触发（满足任一）

1. 工作区有未提交改动（`git status` 非干净）
2. 距离上次 commit > 10 分钟
3. 即将改动的文件与上次 commit 无关（跨任务/跨模块）

### 去重（满足任一时跳过）

1. 上次 commit 是同任务的存档/里程碑/保护快照，且间隔 < 5 分钟
2. 本轮 write-preflight 已建立过保护快照

### Commit 格式

```bash
git add -A
git commit -m "「{任务}#保护」chore: 自动存档"
```

### 输出格式

```
💿 已保护 → a1b2c3d
```

## 💾 存档

存档是每步改动后的细粒度记录，用于逐步回退。

### 各模式存档策略

| 模式 | 触发时机 | 指纹格式 |
|------|----------|----------|
| **Step 模式** | 每步验证通过后 | `「{任务}#{序号}」` |
| **信任模式** | 全部完成后（1个） | `「{任务}#信任完成」` |
| **FastTrack** | 改动验证通过后 | `「{改动}#01」` |
| **Debug/Hotfix** | 修复验证通过后 | `「{问题}#修复」` / `「HOTFIX#{问题}」` |
| **Refactor/Optimize** | 每步等价/性能验证后 | `「{目标}#{序号}」` |
| **Cleanup** | 清理验证后 | `「清理#{日期}」` |
| **Tester** | 测试通过后 | `「{测试}#01」` |

### 指纹规范

- **长度**：≤ 10 个字
- **内容**：业务功能摘要，描述"做了什么功能"
- **唯一性**：`<业务摘要>#<两位序号>`，如 `会员一键登录#01`

| ❌ 技术描述 | ✅ 业务功能摘要 |
|------------|----------------|
| 加了个接口 | 订单批量导出#01 |
| 修复null报错 | 支付超时重试#01 |
| 改了CSS | 商品详情改版#01 |

### Commit 格式

```bash
git add -A
git commit -m "「{指纹}」{类型}: {技术描述}"
```

**类型**：`feat` | `fix` | `refactor` | `perf` | `docs` | `chore`

示例：
```bash
git commit -m "「会员一键登录#01」feat: 添加微信OAuth登录流程"
git commit -m "「支付超时重试#01」fix: 处理支付接口超时后的重试逻辑"
```

### 输出格式

```
💾【存档】{任务}#{序号} → b2c3d4e
```

## 读档（展示存档列表）

读档是一个可复用模块：基于真实 git 数据渲染存档列表，让用户选择回退目标。所有模式在需要展示回退选项时，统一使用本模块。

### 触发词

| 用户说 | AI 行为 |
|--------|---------|
| "回退" / "撤销" | 执行读档流程，展示选项 |
| "能回退吗" / "有哪些存档" | 执行读档流程，展示选项 |
| "出问题了" / "代码写坏了" | 先确认是否需要回退，是则执行读档流程 |
| "全部撤销" / "回到最开始" | 执行读档流程，高亮里程碑 |

### 执行步骤（强制）

**Step 1**：执行 git 命令获取真实数据

```bash
git log --oneline -20 | grep "「"
git tag -l "milestone/*" --sort=-creatordate
```

**Step 2**：对每条 commit 判断类型

```
对每条 commit:
├─ git tag --points-at {hash} 含 milestone/* → 🎯 里程碑
├─ 指纹含 #保护                            → 💿 保护快照
└─ 其他                                     → 💾 存档
```

**Step 3**：智能过滤

```
必定展示: 所有里程碑（🎯）
必定展示: 最近 1 个保护快照（💿，如有）
必定展示: 最近 3 个存档（💾）
可选展示: 用户说"查看更多" → 展示全部
```

**Step 4**：渲染唯一格式，展示给用户

### 输出格式（唯一，所有场景通用）

```
━━━━━━━━━━━━━━━━━━━━
📍 存档列表
━━━━━━━━━━━━━━━━━━━━
[1] 🎯 a1b2c3d「会员登录#起点」
[2] 💿 b2c3d4e「会员登录#保护」
[3] 💾 c3d4e5f「会员登录#01」
[4] 💾 d4e5f6g「会员登录#02」
[5] 💾 e5f6g7h「会员登录#03」
━━━━━━━━━━━━━━━━━━━━
回退到哪个？（输入序号或指纹）
━━━━━━━━━━━━━━━━━━━━
```

### 禁止行为

- ⛔ 禁止不执行 git 命令就凭记忆展示存档列表
- ⛔ 禁止编造不存在的 commit hash

## 回退执行

### 回退方式

用户可以说：
- "回退到「会员注册表单#01」"（用指纹）
- "回退到 b2c3d4e"（用 hash）
- "回退"（触发读档，展示选项让用户选）

### 回退安全检查

```
回退请求
├─ 当前在工作分支
│   ├─ 工作区干净 → 执行回退
│   └─ 工作区有未提交改动 → 先建保护快照「现场保存#NN」，再回退
└─ 当前在受保护分支 → 拒绝，提示切换分支
```

### 执行回退

```bash
git reset --hard {hash}
```

### 回退输出

**在工作分支上回退**：
```
━━━━━━━━━━━━━━━━━━━━
✅ 已回退
当前位置:「{指纹}」{hash}
分支: {工作分支}
━━━━━━━━━━━━━━━━━━━━
```

**在受保护分支上请求回退**：
```
━━━━━━━━━━━━━━━━━━━━
⚠️ 当前在受保护分支 {分支名} 上，无法直接回退

请选择:
[1] 切换到工作分支 {工作分支} 后回退
[2] 取消操作
━━━━━━━━━━━━━━━━━━━━
```

## 里程碑：合并与推送询问

### 触发时机

以下情况**必须询问**用户是否合并/推送：

1. 信任模式任务完成后
2. Step 模式完成一个完整功能后（AI 判断）
3. 用户选择"结束"时
4. AI 感知到阶段性成果完成时

### 询问格式

```
━━━━━━━━━━━━━━━━━━━━
✅ 已完成: {功能描述}
📁 改动: {文件数}个文件

🌿 当前工作分支: {工作分支名}
📍 存档历史:
   🎯 {hash}「{指纹}」
   💾 {hash}「{指纹}」
   💾 {hash}「{指纹}」
━━━━━━━━━━━━━━━━━━━━
📌 下一步:
[1] 合并到 {集成分支} 并推送到远程
[2] 合并到 {集成分支}（不推送）
[3] 继续在工作分支开发
[4] 仅推送工作分支（便于创建 PR）
[5] 保存后结束（暂不合并）
[0] 直接结束（暂不合并）
━━━━━━━━━━━━━━━━━━━━
```

### 合并执行流程

**选择 [1] 合并并推送**：
```bash
git checkout {集成分支}
git merge --squash {工作分支}
git commit -m "「{指纹}」{类型}: {描述}"
git push origin {集成分支}
```

**输出**：
```
━━━━━━━━━━━━━━━━━━━━
✅ 已合并并推送
源分支: {工作分支}
目标分支: {集成分支}
远程: origin/{集成分支} ({hash})
━━━━━━━━━━━━━━━━━━━━
📌 是否删除工作分支？
[1] 删除本地工作分支
[2] 保留工作分支
━━━━━━━━━━━━━━━━━━━━
```

**选择 [2] 仅合并**：
```bash
git checkout {集成分支}
git merge --squash {工作分支}
git commit -m "「{指纹}」{类型}: {描述}"
```

**选择 [4] 仅推送工作分支**：
```bash
git push -u origin {工作分支}
```

**输出**：
```
━━━━━━━━━━━━━━━━━━━━
✅ 已推送工作分支
分支: {工作分支}
远程: origin/{工作分支}
━━━━━━━━━━━━━━━━━━━━
💡 可在 GitHub/GitLab 创建 PR 合并到 {集成分支}
━━━━━━━━━━━━━━━━━━━━
```

## 默认策略

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `push_default` | `false` | 不自动推送，需用户明确选择 |
| `merge_strategy` | `squash` | 合并时 squash，保持主线整洁 |
| `integration_mode` | `merge_allowed` | 允许本地合并；`pr_only` 则只推工作分支 |

## 异常处理

| 情况 | 处理方式 |
|------|----------|
| 无 `.autodev/path.md` | 提示用户先创建，使用模板 `assets/templates/path.md` |
| 当前在受保护分支 | 自动创建工作分支后再建立里程碑 |
| push 失败 | 报告错误，保留本地 commit，提示手动处理 |
| 冲突 | 停止自动流程，提示用户解决冲突后手动处理 |
| 无改动 | 跳过 commit，正常输出任务完成报告 |

## 平台注意事项

### Windows PowerShell

PowerShell 不支持 `&&` 链接命令，必须分开执行：

```powershell
git add -A
git commit -m "「指纹」feat: 描述"
```

### Linux / macOS

可以链接执行：

```bash
git add -A && git commit -m "「指纹」feat: 描述"
```

## 常用命令

### 查看存档历史
```bash
git log --oneline | grep "「"
```

### 查看里程碑
```bash
git tag -l "milestone/*" --sort=-creatordate
```

### 手动回退
```bash
git reset --hard {hash}
```

### 查看当前分支
```bash
git branch --show-current
```

## 禁止行为

- ❌ 在受保护分支上直接建立存档/里程碑
- ❌ 不询问用户就自动合并到集成分支
- ❌ 不询问用户就自动推送到远程
- ❌ 在受保护分支上执行 `git reset --hard`
- ❌ 回退时不保存工作区未提交改动
- ❌ 不执行 git 命令就凭记忆展示存档列表
- ❌ 编造不存在的 commit hash
