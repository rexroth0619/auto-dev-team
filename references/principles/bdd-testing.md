# BDD and Behavior Scenarios

> In this skill, BDD first means a behavior contract, then a `.feature` file.

## Each Scenario Should State

- user goal
- preconditions
- triggering action
- expected result
- failure or error behavior
- boundary conditions
- whether manual validation is required

## What AI May Add

- invalid input
- empty / zero / default cases
- boundary values
- failure prompts
- retry or recovery cases
- legacy-data regression cases

## What AI Must Not Invent Alone

- lockout thresholds
- permission matrices
- timeout / retry policy
- approval policy
- business-critical thresholds
